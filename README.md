# astrbot_plugin_image_toolkit

## 插件简介

`astrbot_plugin_image_toolkit` 是一个适用于 AstrBot 的本地图片处理插件，基于 `Pillow` 提供常见图片编辑能力。插件支持在聊天中直接对图片执行查看信息、尺寸调整、裁剪、旋转、灰度化、模糊、格式转换，以及半边轴对称和整图翻转等操作。

当前版本除了支持普通聊天场景下“附图执行/回复图片执行”外，也兼容通过其他插件或 LLM 工具链传入的图片引用路径。

## 当前支持的图片来源

插件会尽量从以下来源中提取第一张可处理图片：

- 当前消息中的 `Image` 组件
- 当前消息中的图片型 `File` 组件
- 被回复消息中的 `Image` 组件
- 被回复消息中的图片型 `File` 组件
- 本地文件路径
- `file://` URI
- `http://` / `https://` 图片地址

说明：
- 对于远程图片地址，插件会先下载到插件临时目录，再交给 `Pillow` 处理。
- 对于 Windows 本地路径，插件兼容 `C:/...`、`C:\...`、`file:///C:/...` 等常见格式。

## 功能说明

本插件提供以下指令：

### 信息与基础处理
- **img_info**：查看图片详细信息，包括格式、尺寸、大小等。
- **img_resize**：调整图片尺寸，需指定目标宽度和高度。
- **img_crop**：裁剪图片，需指定左上角和右下角坐标。
- **img_rotate**：旋转图片，需指定旋转角度。
- **img_gray**：转换为灰度图。
- **img_blur**：应用高斯模糊，可指定模糊半径。
- **img_convert**：转换图片格式或压缩质量。

### 半边轴对称
以下指令不是“整图翻转”，而是以中轴线为基准，用一半内容镜像覆盖另一半：

- **img_mirror_lr**：以竖直中轴线为参考，将**左半区镜像到右半区**。
- **img_mirror_rl**：以竖直中轴线为参考，将**右半区镜像到左半区**。
- **img_mirror_ud**：以水平中轴线为参考，将**上半区镜像到下半区**。
- **img_mirror_du**：以水平中轴线为参考，将**下半区镜像到上半区**。

### 整图翻转
以下指令才是传统意义上的整张图片镜像翻转：

- **img_flip_lr**：整张图左右翻转。
- **img_flip_ud**：整张图上下翻转。

### 帮助
- **img_help**：查看图片工具指令说明。

## 插件流程

1. **监听指令**
   - 插件通过 `@filter.command` 注册各个图片处理命令。

2. **提取图片来源**
   - 优先从当前消息和回复消息中收集图片组件。
   - 同时兼容图片型 `File`、本地路径、`file://` URI、以及远程图片 URL。

3. **归一化图片路径**
   - 本地路径会做规范化处理。
   - 远程 URL 会先下载到插件临时目录。

4. **执行图片处理**
   - 使用 `Pillow` 加载图片，并根据具体命令执行对应处理。

5. **保存结果图**
   - 处理结果会保存到：
     - `data/plugin_data/astrbot_plugin_image_toolkit/`
   - 文件名使用 UUID，避免覆盖和并发冲突。

6. **返回结果**
   - 通过 `yield event.image_result(save_path)` 将图片结果返回。

## 使用方法

执行图片处理命令时，通常有两种方式：

- **直接发送图片并附带指令**
- **回复一张图片后再发送指令**

如果接入了其他插件或 LLM 工具链，也可以通过其构造的图片引用路径触发处理。

---

### 1. 查看图片信息
- **指令**：`/img_info`
- **示例**：
  - 回复图片：`/img_info`
  - 或附图发送：`/img_info`

### 2. 调整图片尺寸
- **指令**：`/img_resize <宽度> <高度>`
- **示例**：`/img_resize 800 600`

### 3. 裁剪图片
- **指令**：`/img_crop <左上角X> <左上角Y> <右下角X> <右下角Y>`
- **示例**：`/img_crop 100 100 500 500`

### 4. 旋转图片
- **指令**：`/img_rotate <角度>`
- **示例**：`/img_rotate 90`

### 5. 灰度化图片
- **指令**：`/img_gray`

### 6. 模糊图片
- **指令**：`/img_blur [模糊半径]`
- **示例**：`/img_blur 5`

### 7. 格式转换与压缩
- **指令**：`/img_convert <目标格式> [质量]`
- **支持格式**：`png` / `jpeg` / `jpg` / `webp`
- **示例**：
  - `/img_convert png`
  - `/img_convert jpeg 75`

### 8. 半边轴对称
- **左 → 右**：`/img_mirror_lr`
- **右 → 左**：`/img_mirror_rl`
- **上 → 下**：`/img_mirror_ud`
- **下 → 上**：`/img_mirror_du`

### 9. 整图翻转
- **左右翻转**：`/img_flip_lr`
- **上下翻转**：`/img_flip_ud`

### 10. 查看帮助
- **指令**：`/img_help`

## 配置说明

当前版本开箱即用，无需额外配置。

代码中已支持以下默认配置读取：

- `default_output_format`
- `default_convert_quality`
- `default_blur_radius`

如果后续要做成可视化配置，可通过插件配置文件或 WebUI 暴露这些字段。

## 注意事项

1. **依赖要求**
   - 本插件依赖 `Pillow`。
   - 若环境未自动安装，请手动执行：`pip install Pillow`

2. **图片输入要求**
   - 指令执行时必须能拿到图片来源；可以是附图、回复图片、本地路径、或兼容链路传入的图片引用。

3. **远程图片说明**
   - 当前版本支持远程图片 URL。
   - 插件会主动下载远程图片到本地临时目录后再处理。

4. **输出文件保留**
   - 当前版本不会在返回结果后立即删除输出图。
   - 这样做是为了兼容某些上游发送链路，避免“图片刚生成就被删除”导致发送失败。
   - 代价是输出目录中的结果图会逐渐累积，后续建议按时间或数量做清理策略。

5. **性能影响**
   - 超大分辨率图片会消耗较多 CPU、内存和处理时间。

6. **参数合法性**
   - 请传入合理参数，例如宽高、坐标、质量、模糊半径等。
   - 非法参数会返回错误提示，但不应导致插件崩溃。

7. **隐私与安全**
   - 图片处理本身在本地完成。
   - 但如果输入的是远程图片 URL，插件会主动发起下载请求以获取原图。
   - 因此“完全无网络请求”已不再适用于当前版本。
